# Experiment: Packet Capture and Analysis Using Wireshark

## Objective

To capture live network packets using Wireshark and identify different network protocols.

## Tool Used

* Wireshark

## Procedure

1. Installed Wireshark.
2. Selected the active network interface (`wlo1`).
3. Started packet capture.
4. Generated traffic by browsing websites and using ping.
5. Captured packets for approximately one minute.
6. Applied protocol filters.
7. Identified network protocols.
8. Saved the capture as a `.pcapng` file.

## Protocols Identified

### 1. DNS (Domain Name System)

**Purpose:** Converts domain names into IP addresses.

**Examples Observed:**

* `google.com`
* `youtube.com`

### 2. TCP (Transmission Control Protocol)

**Purpose:** Reliable communication between devices.

**Observation:**

* TCP connection establishment using SYN, SYN-ACK, and ACK packets.

### 3. UDP (User Datagram Protocol)

**Purpose:** Fast communication without guaranteed delivery.

**Observation:**

* Observed in DNS and QUIC traffic.

### 4. ICMPv6

**Purpose:** Network diagnostics and connectivity testing.

**Observation:**

* Echo Request and Echo Reply packets generated by ping.

## Packet Details

| Protocol | Purpose                      |
| -------- | ---------------------------- |
| DNS      | Name resolution              |
| TCP      | Reliable data transfer       |
| UDP      | Fast data transfer           |
| ICMPv6   | Network diagnostics and ping |

## Conclusion

The network traffic was successfully captured using Wireshark. Multiple protocols including DNS, TCP, UDP, and ICMPv6 were identified. The capture demonstrated how different network protocols interact during normal Internet communication.
